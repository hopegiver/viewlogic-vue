로그 관리

시스템에서 발생하는 모든 활동과 오류를 추적하고 관리합니다.

로그 유형

  • 시스템 로그: 서버 시작/중지, 설정 변경 등
  • 사용자 활동 로그: 로그인, 로그아웃, 페이지 접근
  • 오류 로그: 애플리케이션 오류, 데이터베이스 오류
  • 보안 로그: 로그인 실패, 의심스러운 접근 시도

로그 확인 방법

  1. 모니터링 메뉴에서 '로그 관리'를 클릭합니다
  2. 확인하고자 하는 로그 유형을 선택합니다
  3. 날짜 범위와 필터 조건을 설정합니다
  4. '검색' 버튼을 클릭하여 로그를 조회합니다

💡 로그 관리 팁

  • 로그는 30일간 보관되며, 이후 자동으로 삭제됩니다
  • 중요한 로그는 별도로 백업하시기 바랍니다
  • 로그 파일이 과도하게 증가하면 시스템 성능에 영향을 줄 수 있습니다

성능 모니터링

시스템의 성능 지표를 실시간으로 모니터링하고 분석합니다.

모니터링 항목

  • CPU 사용률: 서버의 처리 능력 상태
  • 메모리 사용량: RAM 사용 현황과 여유 공간
  • 디스크 사용량: 저장 공간 사용률과 I/O 성능
  • 네트워크 트래픽: 인바운드/아웃바운드 데이터 전송량
  • 데이터베이스 성능: 쿼리 응답시간과 연결 수
  • 동접 사용자 수: 현재 접속한 사용자 수

성능 알림 설정

  1. 성능 모니터링 화면에서 '알림 설정'을 클릭합니다
  2. 각 항목별로 임계값을 설정합니다
  3. 알림 받을 이메일 주소를 입력합니다
  4. 알림 조건과 빈도를 설정하고 저장합니다

⚠️ 성능 경고 기준

  • CPU 사용률: 80% 이상 지속 시 점검 필요
  • 메모리 사용률: 85% 이상 시 메모리 증설 검토
  • 디스크 사용률: 90% 이상 시 용량 확장 필요
  • 응답 시간: 3초 이상 시 성능 최적화 필요
